    Versioning

    MarkBind also supports several handy utilities for versioning your site.

    Archiving a site version

    To archive a specific site version, you can use the markbind archive <version_name> command.

    This builds and archives your current site into the _versions/output/<version_name> directory, while outputting it to the versions/<version_name> subdirectory in the output folder.

    Note that versioning is highly dependent on the baseUrl you specify at the time of archiving. Hence, you should still choose your baseUrl wisely as good practice!

    Rebuilding past versions

    You can rebuild all versions previouslay archived with the markbind rebuild-archive command, or a specific version with markbind archive <version>.

    Removing past versions

    To remove a specific version, you can use the markbind remove-archive command.
